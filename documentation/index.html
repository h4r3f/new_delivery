Folder Structure

"If you take a look at the files and folders generated, it all looks pretty similar to an Ionic 1 application initially. We have the typical Cordova style project structure, just with a couple more files (like tsconfig.json which is for TypeScript and webpack.config.js which is for Webpack).

If you look in the app folder though, things start looking a little bit different:

UPDATE: In the latest version of Ionic 2 the file structure looks a little different to this. The app folder now sits outside of the www folder and there’s a few more .scss files.

Usually in an Ionic 1 application, people would have all of their Javascript files (controllers, services etc.) in one folder, all of the templates in another folder, and all of the styling information would usually be contained in a single app.scss file.

The default structure for an Ionic 2 application organises by functionality, so you have all of the logic, templates and styling for a particular component (in the example above we have a basic page component, a list component, and an item details component) in the one spot. This plays in well with the Angular 2 methodology of having everything be standalone components that can easily be reused in other places or projects. If you wanted to reuse a particular feature, or have lots of people working on the same project, the old Ionic 1 approach can become quite troublesome.

The idea of organising code by feature is not particular to Angular 2 & Ionic 2, in fact people were using and advocating for the feature based approach in Ionic 1 applications, it’s just that most people were doing it the other way. With the way Angular 2 works though, using the feature based structure makes a lot more sense."

For more info read: Ionic 2 First Look Series: Your First Ionic 2 App Explained

Usage

  1. Install Ionic CLI `npm install -g ionic@beta`, please setting Ionic CLI work before next step. More info check: http://ionicframework.com/docs/v2/getting-started/installation/
  2. Download and extract the zip file and navigate to the folder in the Terminal.
  3. Run `npm install && ionic state restore` to install dependencies.
  4. Generate icon and splash for your app: `ionic resources`.
  5. Try: `ionic serve`.
